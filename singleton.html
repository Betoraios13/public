Singleton

Para que vários códigos do seu jogo possam acessar classes específicas como por exemplo: save Game, load Game(dentre outras inúmeras funcionalidades) sem que haja conexão direta entre os respectivos códigos podemos usar um singleton. Enfim, crie um script e o nomeie-o como singleton(não é obrigatório) e apague todos os métodos originais como mostra a próxima imagem:

UsingSystem.Collections;
UsingSystem.Collections.Generic;
UsingUnityEngine;

public classsingleton : MonoBehaviuor
{

}

Agora crie uma variável do tipo static do próprio script e em seguida criaremos um método awake que verificará se há este objeto em uma cena ou não, evitando erros futuros:

UsingSystem.Collections;
UsingSystem.Collections.Generic;
UsingUnityEngine;

public classsingleton : MonoBehaviuor
{
public staticsingleton instancia;

private void Awake()
{
DontDestroyOnLoad(this.gameObject);

if(instancia != null)
{
instancia = this;
}
else
{
Destroy(this.gameObject);
}
}
}

Agora crie um ou vários métodos que deseja ser acessado por qualquer outro script na cena

UsingSystem.Collections;
UsingSystem.Collections.Generic;
UsingUnityEngine;

public classsingleton : MonoBehaviuor
{
public staticsingleton instancia;

private void Awake()
{
DontDestroyOnLoad(this.gameObject);

if(instancia != null)
{
instancia = this;
}
else
{
Destroy(this.gameObject);
}
}
public void Quit()
{
Application.Quit();
}
}

Como exemplo, criei esse método que fecha a aplicação. Então, com o singleton pronto anexe ele a algum objeto na cena (obrigatório). E para invocar o método do singleton basta ir ao script desejado e escrever o'que mostra a próxima figura:

UsingSystem.Collections;
UsingSystem.Collections.Generic;
UsingUnityEngine;

public classexemplo : MonoBehaviuor
{
singleton.instacia.Quit();
}
